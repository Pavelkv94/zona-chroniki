# Проект «Зона: Хроники» — симулятор A-Life

Ты работаешь над симулятором живого мира в сеттинге зоны отчуждения.
TypeScript strict, ECS (bitecs), монорепо: /packages/sim (headless ядро),
/packages/ui (React), /packages/headless (CLI), /packages/shared (типы).

## Нерушимые законы (нарушение = задача провалена)
1. ГЛАВНЫЙ ТЕСТ: перед реализацией любой функции ответь на вопрос
   «Будет ли эта система работать, если игрок исчезнет из мира?»
   Если нет — остановись и предложи другую архитектуру.
2. Причинность вместо случайности. Запрещены проверки вида «X% шанс события».
   События возникают из состояния мира. RNG — только seeded PRNG из core/rng.ts
   и только для физиологии (разброс выстрела) и генерации мира.
3. Всё физически существует. Запрещено создавать предметы/деньги/NPC из воздуха.
   Любое появление предмета обязано иметь источник (найден/создан/добыт/украден).
4. Никаких квестов, скриптовых сцен, бессмертных NPC, спавна возле кого-либо,
   NPC без имени-фамилии, NPC в состоянии idle.
5. /packages/sim не импортирует ничего из DOM/React. Симуляция обязана
   запускаться headless в Node.
6. Системы не вызывают друг друга напрямую. Общение — только через компоненты
   ECS и шину событий (core/events.ts). Каждое событие имеет causedBy.
7. Все балансовые константы — только в /sim/balance. Магические числа в
   системах запрещены.
8. Детерминизм: одинаковый seed → одинаковая история. Запрещены Math.random,
   Date.now, итерация по Map/Set без сортировки ключей внутри симуляции.
9. Каждый новый модуль сопровождается: docblock-объяснением, диаграммой
   зависимостей (mermaid в /docs), примером использования, юнит-тестами.
10. Весь контент (фракции, имена, предметы, шаблоны радио) — в /sim/data JSON.
    Код оперирует абстрактными id.

## Definition of Done для любой задачи
- npm run test проходит; добавлены тесты на новое поведение
- npm run sim:100days проходит без ошибок и деградации метрик здоровья мира
- Нет нарушений законов 1–10
