### Открытые хвосты, которые ОБЯЗАНА закрыть задача 0.5 (сериализация):
- **SoA-компоненты при reuse eid (D-008):** `purgeEntity` чистит только ResourceStore, не SoA bitecs. Сериализовать/восстанавливать только ЖИВЫЕ сущности; не захватывать компоненты освобождённых eid.
- **Восстановление форков PRNG (0.3):** `Rng` отдаёт только `state`; seed подпотока выведен из (label, rootSeed). Снапшот обязан хранить label или seed каждого потока, иначе восстановленный fork не воспроизведёт детей (D-004).
- **Монотонность EventId через save/load (C-4):** seam уже готов — `createEventBus(getTick, { eventSeq, log })` принимает стартовый счётчик и лог. 0.5 обязана их сохранять/восстанавливать.
